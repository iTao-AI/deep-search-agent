# 基于 DeepAgents 的企业级深度搜索与研究智能体系统

## 1. 项目在企业当中的价值

本项目旨在解决企业在复杂决策场景下面临的**信息孤岛**与**研究效率低下**痛点。传统 RAG 系统往往只能单次检索，难以处理跨源、多步骤的复杂任务。本系统通过构建“深度搜索研究员”，实现了以下核心价值：

- **多源数据融合决策**：打破了公开互联网信息（Tavily）、企业内部结构化数据（MySQL）与私有非结构化文档（RAGFlow）之间的壁垒，为管理层提供全方位的数据支撑。
- **自动化报告生成**：将原本需要人工数小时完成的“搜索 - 整理 - 撰写 - 排版”流程，自动化为分钟级的端到端输出（支持 Markdown/PDF），显著降低人力成本。
- **可信赖的执行机制**：引入人机协作（HITL）与工具审批机制，确保涉及数据库修改或文件删除等高危操作时的安全性，满足企业合规要求。
- **知识沉淀与复用**：通过长期记忆模块（StoreBackend），实现跨会话的知识共享，避免重复劳动，让智能体具备“持续成长”的能力。

## 2. 项目架构

本项目采用“主智能体统筹 + 多专家子智能体并行协作”**的分布式架构，基于 DeepAgents + LangGraph** 构建状态化工作流，后端采用 **FastAPI** 异步服务，前端通过 **WebSocket** 实现实时交互。

- **交互层**：Vue3 前端 + FastAPI 后端，通过 WebSocket 全双工通信，实时推送智能体的思考过程（Thinking Process）和工具执行状态。
- **编排层（Core）**：
  - **Main Agent（项目经理）**：负责任务拆解、规划（Planning）、资源调度及最终报告汇总。
  - **Sub-Agents（专家团队）**：
    - **网络搜索助手**：负责广域公开信息检索（Tavily），支持多轮递进搜索。
    - **数据库查询助手**：负责企业内部结构化数据查询（MySQL），支持 Text-to-SQL 与安全执行。
    - **RAGFlow 助手**：负责私有知识库检索，解决内部规章与技术文档查询。
- **工具与存储层**：
  - **工具集**：自定义开发搜索、SQL 执行、文件读写、PDF 转换等工具。
  - **存储后端**：采用 **CompositeBackend** 混合存储策略，临时文件存本地，重要记忆存数据库（KV Store），实现上下文管理与持久化记忆分离。
- **基础设施**：使用 **ContextVars** 实现异步并发下的会话隔离，确保多用户请求数据不串线。

## 3. 项目亮点

- **深度规划与自我修正**：基于 DeepAgents 的 `write_todos` 机制，智能体不仅能拆解任务，还能在执行过程中根据反馈动态调整计划（如搜索无果时自动切换关键词），具备闭环纠错能力。
- **上下文溢出解决方案**：针对长任务导致的 Context Window 限制，设计了**文件系统卸载机制**。智能体将中间结果写入外部存储（本地/DB），按需读取，有效支撑长链路任务。
- **安全可控的人机协作（HITL）**：针对高危工具（如删库、删文件），实现了中断 - 审批 - 恢复流程。支持人工对工具参数进行**编辑修正**后再执行，极大提升了系统安全性。
- **全链路可观测性**：自研 **Middleware 与 Monitor 系统**，通过 WebSocket 实时向前端推送每一步的工具调用、参数、耗时及思考内容，解决了 Agent 黑盒问题，便于调试与监控。
- **技能热插拔机制**：基于 `SKILL.md` 构建技能库，支持动态加载领域知识（如代码审查、数据清洗），无需修改核心代码即可扩展智能体能力。

## 4. 项目难点与解决方案

- **难点一：异步并发下的会话隔离**
  - *问题*：FastAPI 异步处理多用户请求时，全局变量会导致用户数据（如文件路径、Thread ID）串混。
  - *解决*：引入 Python **`ContextVars`** 上下文变量，为每个请求链路绑定独立的 `thread_id` 和 `session_dir`，确保深层工具调用能“隔空取物”且互不干扰。
- **难点二：Agent 幻觉与工具执行失败**
  - *问题*：模型可能臆造数据库表名或生成错误 SQL，导致执行报错。
  - *解决*：在 Prompt 中强制规定**“三步走”策略**（先查表结构 -> 再预览数据 -> 最后执行 SQL），并在工具层增加安全清洗（如移除 SQL 注入字符）与异常捕获，返回友好错误信息引导模型重试。
- **难点三：长任务的状态持久化**
  - *问题*：任务执行时间长，服务器重启或超时会导致状态丢失。
  - *解决*：集成 **LangGraph Checkpointer**，将执行状态持久化到存储后端。支持中断后通过 `thread_id` 恢复现场，确保任务可断点续传。
- **难点四：复杂文件路径管理**
  - *问题*：跨平台（Windows/Linux）路径分隔符差异及虚拟路径映射问题。
  - *解决*：封装统一的 `path_utils` 模块，基于 `pathlib` 实现路径解析，自动清洗虚拟前缀（如 `/workspace`），并防止路径嵌套与越权访问。
