# Security Policy

## Reporting A Vulnerability

Do not disclose suspected vulnerabilities in public Issues or pull requests.

Use GitHub private vulnerability reporting for this repository. Include the affected behavior, reproduction steps, expected impact, and any suggested mitigation.

## Scope

Security reports should concern repository code, dependencies, public contracts, or documented runtime behavior. Treat uploaded files, model output, tool responses, external service responses, and generated reports as untrusted inputs.
